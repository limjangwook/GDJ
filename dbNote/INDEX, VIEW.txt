* INDEX
 - 빠른 검색을 위해 특정 칼럼에 인덱스를 등록
 - PK나 UNIQUE를 가진 칼럼은 인덱스가 자동으로 등록된다
 - CREATE INDEX 인덱스_이름 ON 테이블_이름(칼럼_이름);   -> 비고유 인덱스 생성법

 -CODE는 PK이고, VARCHAR2타입-

  WHERE CODE = '1' (O) 
  WHERE CODE = 1 (동작함)
   └> WHERE TO_NUMBER(CODE) = 1(내부동작) 
        : 자동캐스팅 되어서 동작 가능
        : But, TO_NUMBER()함수로 인해 CODE의 인덱스를 사용할  수 없음. PK니까 어차피 인덱스가 세팅되어있어 빠르게 조회가 되는데 함수 개입으로 인해 인덱스를 못 쓰게 됨. -> 성능도 떨어지고 느려져서 좋지 않다

 - WHERE절 작성 시 가급적 등호 왼쪽은 가공 안 하는 것이 좋다. (TO_NUMBER(CODE)) >> 조건을 검색할 때 인덱스를 사용할 가능성이 높기 때문에. 


* VIEW
 - 가상테이블
 - 디스크에 저장된 테이블이 아니고 쿼리만 저장해 둔 형태
 - 자주 쓰는 복잡한 쿼리문이 있으면 뷰로 저장해 둠
 - 저장된 뷰는 테이블처럼 SELECT문을 활용하면 된다.
 - DBA 권한이 있어야 생성 가능
 - CREATE VIEW 뷰_이름 AS (SELECT~ 쿼리문)
